# Version pinning strategy:
#   1. Start with the highest Python version we support and pin the latest available package
#      version for it. If that version also supports older Python versions, it covers a range
#      (e.g. "python_version >= X").
#   2. Work backwards through older Python versions, pinning the highest package version that
#      is still compatible with each one.
#
# This means that when a new Python version is added, only the last (open-ended range) entry
# for each package typically needs to be updated.
